### 执行摘要

企业政策洞察平台（EPIP）是一个 AI 驱动的知识图谱（KG）系统，专为香港公开政策数据集（如 data.gov.hk 的健康统计、预算和法规）设计。该平台以本地部署确保数据隐私，通过 RAG 和 GraphRAG 技术实现多步推理、最小化幻觉和可追溯输出。基于提供的文档细节，本白皮书聚焦核心实现：使用 Light-RAG 构建 KG 并手动优化，随后强调查询处理的多步推理和幻觉检测机制。短期目标为发表小论文（如“基于 Light-RAG 的异构 CSV KG 在政策查询中的应用”），长期探索商业化路径。

---

### 1. 引言

#### 1.1 项目背景
EPIP 从 Policy-RAG 演进，针对异构 CSV 数据（如 Year、Statistic_Type、Value 等列）构建 KG。文档强调基础稳固：使用 Light-RAG（一个轻量级 GraphRAG 框架，支持知识图谱构建和检索）进行初始 KG 构建，随后手动优化实体关系。查询处理聚焦中心节点的多步推理，支持实时企业查询（如医疗影响链）。幻觉检测确保输出可靠。该方案避免目标过大，优先小突破。

#### 1.2 目标
- **短期**：基于下载数据集（如主要健康统计）构建优化 KG；实现实体准确率 >90%、关系覆盖率 >80%；扩展查询和检测功能。
- **长期**：商业化为政策决策工具。
- **关键指标**：查询速度提升 2-5 倍、幻觉率 <5%、数据完整率 >95%。

#### 1.3 数据集
优先结构化 CSV：
1. 主要健康相关统计。
2. 入院统计。
3. 香港国内健康账户。
4. 食品安全和公共卫生统计。
5. ICAC 年度预算。
6. 家庭支出调查结果。
已全部下载到dataset文件夹

### 2. 系统架构

EPIP 为闭环架构：数据准备 → KG 构建 → 查询处理 → 幻觉检测 → 输出。聚焦 Light-RAG 驱动的 KG 和 ReAct 查询。

#### 2.1 高层概述
- **输入**：异构 CSV。
- **核心**：Light-RAG KG 构建 + 多步推理。
- **输出**：可追溯洞察和可视化。
- **部署**：本地 Neo4j，支持多租户。

#### 2.2 关键组件
- **KG 构建**：Light-RAG + 手动优化。
- **查询处理**：中心节点推理。
- **幻觉检测**：多维度验证。
- **优化**：Redis 缓存、D3.js 可视化。

### 3. 技术细节

#### 3.1 开发环境
- **语言**：Python 3.10+。
- **库**：pandas、numpy、sentence-transformers、neo4j、Light-RAG（GitHub 开源框架，如 HKUDS/LightRAG，支持图构建和 API）、openai（封装阿里云通义千问 API 调用）。
- **数据库**：Neo4j。
- **设置**：虚拟环境；Docker。
- **LLM**：本地 Ollama；阿里云通义千问（qwen-plus）。
- **嵌入**：本地 sentence-transformers；阿里云 text-embedding-v4。

#### 3.2 数据准备
清洗 CSV：验证格式、填充缺失、标准化术语。增量处理，确保质量 >95%。

#### 3.3 知识图谱构建
使用 Light-RAG 进行构建，然后手动优化：
- Light-RAG 初始管道：从 CSV 生成实体嵌入和关系三元组，支持知识图谱探索。
- 手动优化：调整分块（行级 + 语义分组）、实体识别（列映射 + 置信 >0.5）、消歧（Jaccard >0.8）、关系提取（覆盖 >80%）、子图融合（实体对齐，余弦 >0.85）。
- 中文 Prompt 优化：针对 qwen 模型设计实体/关系提取提示词，强调中文政策语境和数字指标描述，降低误报。
- 中文实体类型：政策、组织、人物、地点、日期、指标、疾病、预算、法规、项目、设施、服务。

#### 3.4 查询处理：从中心节点的多步推理
- **输入解析**：使用大模型提取中心节点/实体。
- **图查询**：Cypher 最短路径；优化 Neo4j GDS A* 加速。通过 ReAct 循环多步推理（分解 3-5 子查询，聚合分支）。
- **框架**：
  - **预处理**：解析查询，剪枝无关子图（相似度 >0.7），准备启发值。
  - **执行**：多层引擎——层1: 基本路径（A* 或 Dijkstra，Cypher 集成 GDS.gds.shortestPath.astar）；层2: 多步推理（ReAct 循环，每个子查询用 A*）；层3: 聚合（排名路径，基于置信/长度）。深入：Redis 缓存热门路径（TTL 1 小时），Neo4j EXPLAIN 动态调整。
  - **后处理**：过滤低置信路径 (<0.8)，添加元属性溯源，超时 fallback 到基本路径。
- **优化**：索引加速；并行批量。支持 TB 级图，速度提升 2-5 倍，示例：医疗影响链 (“疫情政策 → 疫苗分发 → 医院资源”)。

#### 3.5 幻觉检测与可追溯性：确保可靠输出
生成后验证事实一致性。
- **检测**：多维度检查（如语义匹配）。
- **可追溯性**：附加路径（如“基于节点 X 来自文档 Y”）。结构化日志跟踪多步路径（VeriTrail-like：步骤带置信度、关键节点/文档）。
- **优化**：D3.js 可视化推理树；集成高级追踪（如 KG 嵌入路径可视化），量化贡献（attention 机制节点权重）。过滤弱点 (>0.7 阈值)；子图元属性增强。

#### 3.6 部署与多租户
Docker 化 Neo4j；RBAC 权限；Prometheus 监控。

#### 3.7 数据导入工具
scripts/import_data.py 负责批量导入 CSV、生成嵌入并写入 Neo4j/Light-RAG 存储，支持增量重跑。
- 断点续传：维护 import 状态文件，重启后从最近成功批次继续。
- 错误重试：对失败批次自动重试 5 次，使用指数退避（2^n 秒）避免 API 限流。
- 命令行参数：`--status` 查看当前批次；`--reset` 清除状态重新导入；`--retry-failed` 仅重跑失败批次；`--max-retries` 自定义最大重试次数。
